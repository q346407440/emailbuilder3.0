# 邮件变量与绑定真源

> 机器校验：`src/payload-contract/`、`src/lib/validate.ts`、`src/lib/merge.ts`（`mergeTemplatePayload`）。  
> Agent 索引：技能 **`easy-email-payload-contract`**、**`easy-email-concepts`**。  
> 本文不维护字段白名单，只说明**同一业务事实写在哪一层**。

## 1. 总原则

**禁止在多处手写等价的槽目录、列表 schema 或业务默认值。** 历史模板曾把 collection 的 `itemFields`、整表 `defaultValue` 写在多个叶子 `bindings` 上；现已统一收敛，新模板与迁移均不得回退。

| 业务事实 | 唯一真源 | 模板里只保留什么 |
|----------|----------|------------------|
| 变量槽目录（label、valueType、collection 列定义、min/max、dataSource） | **`payload.json` → `slots`** | `bindings` / `repeat` / `visibility` 上的 **`slotId` + `mode` + `valueType`（及 `slotPath` 等绑定关系）** |
| 变量当前取值 | **`payload.json` → `values`** | **不写** `bindings.*.defaultValue` |
| collection 列表列 schema（`itemFields` 等） | **`blocks.*.repeat`**（或极少数 **`visibility`**）；与 **`payload.slots[slotId]`** 对齐 | 行模板 binding 仅 **`slotPath: "0.<fieldKey>"`**，**不**重复 `itemFields` / `minItems` / `maxItems` / 整表 `defaultValue` |
| 列表行结构（原型子树） | **`template.json`** block 树 + **`repeat.prototypeChildIds`** | — |
| 主题字段「解除跟随」快照 | **`template.meta.easyEmailBindingUi`**（仅编辑器） | 与渲染/合并无关；无 detached 字段时应**无**此 meta |
| 画布预览合并 | **`mergeTemplatePayload`** 只读 **`payload.values`** | 不读 binding `defaultValue` |

从 template 扫描合并槽目录的实现：`src/payload-contract/slot-registry.ts`（`buildExternalSlotRegistry`）；场景多版式用 `buildUnionExternalSlotRegistry`。日常维护与变量面板以 **`payload.slots`** 为准（`src/lib/payloadSlots.ts` · `collectPayloadVariableSlots`）。

## 2. `template.bindings` 上允许写什么

### 2.1 标量 / URL / 图片等 variable 绑定

```json
{
  "slotId": "heroTitleText",
  "mode": "variable",
  "valueType": "string",
  "allowExternal": true,
  "fieldKind": "content"
}
```

**禁止**：`defaultValue`、`label`、`description`（目录在 **`payload.slots`**；取值在 **`payload.values`**）。

### 2.2 列表 repeat 行内字段

repeat 宿主示例（`layout` / `grid`）：

```json
"repeat": {
  "mode": "collection",
  "slotId": "memberBenefits",
  "prototypeChildIds": ["mwc-benefit-row"],
  "fallbackChildIds": ["mwc-benefit-row"],
  "itemFields": [ { "key": "title", "label": "…", "valueType": "string", "required": true } ],
  "minItems": 5,
  "maxItems": 5,
  "label": "会员权益列表",
  "description": "…"
}
```

行模板内 binding 示例：

```json
{
  "slotId": "memberBenefits",
  "mode": "variable",
  "valueType": "collection",
  "allowExternal": true,
  "fieldKind": "content",
  "slotPath": "0.title"
}
```

**禁止**在行模板 binding 上再写：`itemFields`、`minItems`、`maxItems`、`defaultValue`（整表数组）、与 repeat 重复的 `label`/`description`。

**禁止**在静态多行上写 `1.title`、`2.imageSrc` 等下标（应使用 repeat；校验见 `validateTemplateBindings` warning/error 策略）。

### 2.3 interpolate 原子槽

`bindings` 上保留 `interpolationSlots[]`（含 `slotId`、`valueType`、`allowExternal`）。  
**禁止**在原子槽上写 `defaultValue`；缺省展示用 **`payload.values[slotId]`**。

### 2.4 theme 绑定

`mode: "theme"` + `tokenPath`：不受 payload 槽目录约束，仍按 **`token-preset-contract`**。

## 3. `payload.json` 形态

```json
{
  "schemaVersion": "1.0.0",
  "slots": {
    "heroTitleText": {
      "label": "主标题",
      "valueType": "string"
    },
    "memberBenefits": {
      "label": "会员权益列表",
      "valueType": "collection",
      "itemFields": [ … ],
      "minItems": 5,
      "maxItems": 5
    }
  },
  "values": {
    "heroTitleText": "Welcome",
    "memberBenefits": [ { "title": "…", "subtitle": "…", "iconSrc": "https://…" } ]
  }
}
```

- **`slots`**：变量目录真源（与 template 中引用的 `slotId` 集合应对齐；校验 `validatePayloadAgainstTemplate` / `validateTemplateReferencesPayloadSlots`）。
- **`values`**：赋值真源；预览合并唯一输入（`src/lib/merge.ts`）。
- **`detachedVariableSlotIds`**（可选）：解除外部赋值的槽，合并时跳过。

新建邮件或增删槽时：先改 template 绑定关系 → 同步 **`payload.slots`** → 填 **`payload.values`**。可用 **`npm run migrate:payload-slots:write`** 从 template 合并生成/补齐 `slots`。

### 3.1 列表行字段 `itemFields`（契约真源）

机器实现与枚举：**`src/payload-contract/collection-item-fields.ts`**（类型展示/归一、嵌套深度）+ **`value-types.ts`**（`COLLECTION_ITEM_FIELD_TYPES`）。**禁止**在组件、技能或 docs 中另维护一份类型白名单。

| 规则 | 说明 |
|------|------|
| 可选 `valueType` | 与标准标量一致：**文本 / 数值 / 链接**，再加 **列表**（`collection`）；**不**单独区分「图片」，图片 URL 用 **链接** |
| 存量 `image` | 校验仍接受；保存/UI 确定时应 **`normalizeCollectionItemFields`** 归一为 `url` |
| 列表层级 | **最多 2 级**：槽位列表 + `itemFields` 内一层子列表字段；子列表的 `itemFields` **不得**再含 `collection` 类型 |
| 声明位置 | `payload.slots[slotId].itemFields` 与 **`blocks.*.repeat.itemFields`** 对齐；行模板 binding 只写 `slotPath`，不重复 `itemFields` |

## 4. `template.meta`（仅编辑器）

`meta.easyEmailBindingUi` 仅存 **主题字段解除跟随** 时的恢复快照（`themeRestoreJson` / `themeRestoreBindingJson`），见 `src/lib/bindingUiMeta.ts`。  
**不是**业务默认值存储；空对象 `{}` 应删除。

## 5. 存量清理迁移（2026-05）

| 脚本 | 作用 |
|------|------|
| `npm run migrate:static-collection-to-repeat:write` | 静态多行/多格 → `repeat` |
| `npm run migrate:prune-collection-binding-meta:write` | 去掉 repeat 已声明后叶子 binding 上的 collection 元数据 |
| `npm run migrate:prune-scalar-binding-default-value:write` | 去掉标量 / interpolate 原子槽上的 `defaultValue` |
| `npm run migrate:payload-slots:write` | 从 template 合并补齐 `payload.slots` |

交付前：**`npm run validate:all`**（或日常 `npm run validate:email`）。

## 6. 变更检查清单

- [ ] 新 collection 槽：`repeat.itemFields` + `payload.slots` + `payload.values` 三者一致，叶子 binding 无重复 schema。
- [ ] 新标量槽：`payload.slots` 有目录项，`payload.values` 有值，binding 无 `defaultValue`。
- [ ] 未新增「仅存在于 binding、不在 payload.slots」的 label 真源。
- [ ] 全量迁移后已跑 `validate:all`，相关模板无 collection 下标 warning（若仍保留 warning 级校验，见 `blockingValidationIssues` 与 server 落盘策略）。

## 7. UI 用语：列表 vs 数组

| 场景 | 推荐用语 |
|------|----------|
| 侧栏、Inspector、弹窗、按钮（`valueType === "collection"`） | **列表**（列表变量、列表重复、列表行字段、列表项） |
| JSON 粘贴、解析失败、契约校验报错（描述数据结构） | 可保留 **数组**（如「JSON 根节点须为数组」「collection 槽的值必须为数组」） |
| 代码标识符 | 保持 `collection`、`itemFields`、`values[]` 等英文 |

- 槽级 `valueType` 展示名：`src/payload-contract/value-type-labels.ts` · `payloadSlotValueTypeLabel()`。
- **列表行字段** `itemFields[].valueType` 展示名：`collection-item-fields.ts` · `collectionItemFieldValueTypeLabel()`。

勿再使用「数组列表」「模板默认」指代 `payload.values` 中的已存取值。
